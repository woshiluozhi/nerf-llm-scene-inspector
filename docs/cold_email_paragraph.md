## Technical Version

I recently built NeRF-LLM Scene Inspector, a research engineering system for open-vocabulary 3D scene inspection from phone video or image captures. It wraps Nerfstudio-style data processing and baseline NeRF training, uses LERF-style language-field querying as the primary semantic backend, and adds a deterministic local query planner that expands object, material, affordance, and spatial-relation prompts into visual backend calls. The project includes typed QueryResult schemas, multi-view dry-run render artifacts, fallback viewer import/repair workflows for upstream LERF version differences, spatial reasoning heuristics, scene-relation edge reports from query boxes/points, experiment-matrix ablation summaries, paper-style research reports, real-run action plans, claim audits, claim-calibrated submission packets, annotation-based evaluation, environment diagnostics, CI, and professor-facing portfolio documentation. I am not presenting it as a new NeRF architecture or state-of-the-art segmentation method; the goal is to demonstrate rigorous implementation and reproducible experimentation around language-queryable neural scene representations. I would be excited to discuss how systems like this could be extended toward embodied AI, robotics, or persistent physical scene understanding.
